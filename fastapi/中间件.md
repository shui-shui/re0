async / await 深度讲解（重点）：
同步（sync）
response = call_next(request)
必须等这个执行完  整个服务器被“堵住”（在执行期间做不了任何事）
异步（async）：
为什么 FastAPI 强制 async？👉 因为它是：高并发框架（很多人同时访问）
async = 这个函数支持“并发处理”
await = 遇到耗时任务先让出执行权

⚠️ 思考题（进阶）

5️⃣ 为什么中间件函数必须是 async + await？
接口处理是“要等结果”的
比如：
数据库查询（要时间）
调用其他接口（要时间）
👉 await 的作用：
“等这个操作执行完，再继续往下走”
👉如果不用 async：
会卡住整个程序 ❌
性能很差 ❌
📌 简单理解：
async = 这个函数是“会等待的函数”
await = “在这里等一下结果”

6️⃣ 写一个中间件，实现：打印接口执行耗时（简单版）
import time 
from fastapi import FastAPI,request

app=FastAPI()

@app.middleware("http")
async def timer_middleware(request:Request,call_next):
    start=time.time()
    response = await call_next(request)
    end=time.time()
    print("接口耗时:", end - start)
    return response
